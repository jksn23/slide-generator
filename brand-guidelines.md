# **brand guidelines.md — Brand Guidelines untuk Aplikasi Slide Ibadah Generator**

## **1. Nama Merek (Brand Name)**

### **Nama Merek Utama**

Gunakan nama merek:

# **LiturgiSlide**

### **Alternatif Nama**

Jika ingin nama yang lebih lokal dan langsung mencerminkan fungsi aplikasi, gunakan salah satu opsi berikut:

1. **Slide Ibadah Generator**
2. **IbadahSlide**
3. **AutoSlide Ibadah**
4. **GMIM Slide Builder**
5. **Syaloom Slide Generator**

### **Rekomendasi Final**

Gunakan **LiturgiSlide** sebagai nama aplikasi utama.

**Alasan:**

* Singkat dan mudah diingat.
* Langsung menggambarkan fungsi aplikasi: membuat slide dari tata liturgi/ibadah.
* Cocok untuk penggunaan gereja, pelayanan multimedia, dan operator ibadah.
* Tidak terlalu teknis, sehingga mudah dipahami oleh pengguna non-programmer.

---

## **2. Identitas Visual Umum**

Aplikasi ini adalah aplikasi desktop untuk membantu pelayan multimedia gereja membuat slide ibadah secara semi-otomatis dari dokumen Word.

Identitas visual harus mencerminkan:

* **Rapi**
* **Tenang**
* **Modern**
* **Mudah digunakan**
* **Bernuansa pelayanan gereja**
* **Tidak terlalu ramai**
* **Tidak terlihat seperti aplikasi teknis yang rumit**

Tampilan aplikasi harus membuat pengguna merasa bahwa proses membuat slide ibadah menjadi lebih **cepat**, **teratur**, dan **tidak melelahkan**.

---

## **3. Penggunaan Logo (Logo Usage)**

### **Konsep Logo**

Logo aplikasi disarankan menggunakan kombinasi simbol:

* **Ikon slide/presentasi**
* **Ikon salib sederhana**
* **Ikon dokumen Word menuju PowerPoint**
* **Ikon cahaya atau sinar lembut**

Logo tidak boleh terlalu kompleks. Gunakan bentuk sederhana agar tetap jelas di ukuran kecil.

### **Penempatan Logo**

Logo harus ditempatkan pada:

1. **Header aplikasi**

   * Posisi kiri atas.
   * Diikuti teks nama aplikasi **LiturgiSlide**.

2. **Splash screen / loading screen** jika dibuat.

   * Logo berada di tengah.
   * Nama aplikasi berada di bawah logo.

3. **About dialog**

   * Logo berada di bagian atas.
   * Diikuti deskripsi singkat aplikasi.

### **Ukuran Logo**

* Header aplikasi: **32px–40px**
* Splash screen: **96px–128px**
* About dialog: **64px–96px**

### **Aturan Penggunaan Logo**

* Jangan menaruh logo di atas background yang terlalu ramai.
* Jangan menggunakan warna logo yang bertabrakan dengan background.
* Jangan memiringkan, meregangkan, atau memberi efek berlebihan pada logo.
* Gunakan versi logo terang untuk background gelap.
* Gunakan versi logo gelap untuk background terang.

## **5. Tipografi (Typography)**

### **Font Utama Aplikasi**

Gunakan font utama:

* **Segoe UI**

Alasan:

* Cocok untuk aplikasi desktop Windows.
* Terlihat modern dan bersih.
* Mudah dibaca.
* Tersedia secara default di banyak sistem Windows.

### **Font Alternatif**

Jika **Segoe UI** tidak tersedia, gunakan fallback:

* **Arial**
* **Calibri**
* **Inter**
* **Sans-serif**

### **Font untuk Preview Slide**

Untuk preview gaya slide ibadah, gunakan font:

* **Arial Rounded MT Bold**
* **Arial**
* **Segoe UI Semibold**

Jika ingin mendekati tampilan slide ibadah yang tebal dan mudah dibaca dari jauh, gunakan:

* **Arial Rounded MT Bold**

Namun untuk UI aplikasi, tetap gunakan **Segoe UI**.

## **8. Gaya Visual Preview Slide**

Preview tidak harus menampilkan PowerPoint sepenuhnya, tetapi harus memberi gambaran isi slide.

### Preview Card

Setiap slide dalam preview list harus memiliki:

* Nomor slide di kiri
* Badge tipe slide
* Judul/isi singkat
* Checkbox include/skip
* Tombol kecil **Edit** jika fitur edit tersedia

### Contoh Item Preview

```text
[Slide 12] [Liturgi]
P : Jemaat yang dikasihi TUHAN, marilah kita sujud menyembah...
J : Sebab Dialah Allah kita...
```

### Prinsip Preview

* Jangan tampilkan teks terlalu panjang dalam list.
* Batasi preview ke 2–3 baris.
* Berikan opsi expand/detail jika diperlukan.
* Gunakan spacing cukup agar tidak melelahkan mata.

## **13. Prinsip Desain UI/UX**

Agen AI pembuat kode harus mengikuti prinsip berikut:

1. **Prioritaskan kemudahan penggunaan**

   * Pengguna harus bisa memahami aplikasi dalam 1–2 menit.

2. **Minimalkan jumlah klik**

   * Alur utama harus sederhana:
     **Pilih File Word → Preview → Generate PowerPoint**

3. **Berikan feedback yang jelas**

   * Setiap aksi harus menghasilkan status atau pesan.

4. **Jangan membuat UI terlalu ramai**

   * Gunakan panel dan spacing yang cukup.

5. **Gunakan bahasa Indonesia**

   * Semua teks aplikasi harus menggunakan Bahasa Indonesia.

6. **Jangan menampilkan istilah teknis pada pengguna**

   * Istilah teknis boleh ada di kode, tetapi tidak di UI utama.

7. **Gunakan warna aksen seperlunya**

   * Warna kuning/gold hanya untuk highlight penting, bukan background utama.

8. **Pastikan kontras teks tinggi**

   * Teks harus mudah dibaca, terutama pada preview slide.

## **16. Do and Don’t**

### **Do**

* Gunakan **LiturgiSlide** sebagai nama aplikasi.
* Gunakan warna utama **#1F4D3A**.
* Gunakan font **Segoe UI**.
* Gunakan UI sederhana berbasis panel.
* Gunakan bahasa Indonesia yang jelas.
* Berikan preview sebelum generate.
* Berikan pesan status setiap proses.

### **Don’t**

* Jangan membuat tampilan terlalu ramai.
* Jangan menggunakan terlalu banyak warna.
* Jangan menggunakan istilah teknis pada UI.
* Jangan membuat pengguna harus membuka banyak halaman.
* Jangan menaruh tombol utama di tempat tersembunyi.
* Jangan menggunakan font dekoratif untuk UI aplikasi.
* Jangan membuat preview sulit dibaca.
